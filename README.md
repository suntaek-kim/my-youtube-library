# 나의 유튜브 서재

내가 사랑하는 유튜브 영상들을 한 권의 책처럼 모아두는 개인 큐레이션 사이트.
누구나 둘러볼 수 있고, 회원가입하면 자기만의 책장을 가질 수 있어요.

## 🌐 라이브 사이트

### ▶ https://suntaek-kim.github.io/my-youtube-library/

회원가입 없이도 바로 둘러볼 수 있어요.

## ✨ 주요 기능
- 📚 카테고리별 영상 카드 그리드 (책장 무드)
- 🔍 실시간 검색 — 제목·채널·코멘트·카테고리 통합 (200ms 디바운스)
- 🔐 회원가입 / 로그인 (Supabase Auth)
- ☁️ 클라우드 저장 — 어느 기기에서 접속해도 같은 책장
- ⚡ 실시간 동기화 — 다른 기기에서 바꾸면 새로고침 없이 갱신
- 💾 JSON 백업 / 복원
- 🎨 Wanted Design System 기반 (Pretendard Variable + #0066FF)
- 📱 반응형 디자인 (모바일 1열 → 태블릿 2열 → 데스크톱 3열)

## 🚀 사용 방법

### 👀 방문자 (비로그인)
URL 접속만으로 사이트 주인장의 책장을 둘러볼 수 있어요.
영상 구경, 카테고리 필터, 검색, 클릭해서 재생 — 모두 가능해요.

### 🙋 사용자 (회원가입 후)
헤더 우측 **"로그인 / 회원가입"**으로 가입하면 자기만의 책장이 생겨요.
- ➕ 영상 추가 / ✏️ 편집 / 🗑️ 삭제
- 📤 JSON 내보내기 / 📥 가져오기
- 모든 변경은 클라우드에 저장 → 다른 기기에서도 실시간으로 반영돼요

## 🛠️ 기술 스택
- **프론트엔드**: HTML · CSS · JavaScript (프레임워크 없는 바닐라 구성)
- **백엔드**: Supabase — Auth(인증) · Database(PostgreSQL) · Realtime(실시간)
- **호스팅**: GitHub Pages
- **디자인**: Wanted Design System (Pretendard Variable)

## 🔒 보안 모델
- 코드에 포함된 Supabase **anon key는 공개용 키**입니다. 공개돼도 안전하게 설계됐어요.
- 실제 데이터 보호는 **Supabase RLS(Row Level Security) 정책**이 강제합니다:
  - **읽기**: 누구나 모든 영상을 조회 가능 (공개 책장)
  - **추가 / 수정 / 삭제**: 로그인한 사용자가 **자기 영상에 대해서만** 가능
- `service_role` 키는 코드에 포함하지 않습니다.

## 💭 회고

2026년 5월 15–17일, 사흘간의 여정.

HTML 파일 하나로 시작했다. 모든 코드가 뒤엉킨 프로토타입.
그게 파일로 나뉘고(Sprint 1), Git에 올라가고(2), 디자인을 입고(3),
검색이 생기고(4), 로그인이 붙고(5), 데이터가 클라우드로 옮겨가
끝내 실시간으로 동기화됐다(6).

웹 개발이 처음이었다. `defer`도 `localStorage`도 모르던 데서 시작해,
RLS 정책을 검증하고 데드락을 디버깅하는 데까지 왔다.

막히는 곳마다 — `file://` 보안 오류, 인증 데드락, 꺼져 있던 RLS 스위치 —
멈춰 서서 원인을 찾고 고쳤다. 그게 전부였다.

이제 집에서 더한 영상이 회사 화면에 새로고침 없이 떠오른다.
프로토타입이, 진짜 서비스가 됐다.
